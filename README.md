## 一、产品愿景与闭环

VestiCore 的完整产品愿景包含「**风格灵感 → 衣柜匹配 → 虚拟试穿 → 穿搭决策**」闭环。
本次 Demo（2.5 周 · 4 人）聚焦最小可行演示，其中 **AI 品类识别** 与 **虚拟试穿** 为不可妥协的核心亮点。

***

## 二、用户旅程地图（12 个步骤完整闭环）

| 阶段           | 步骤   | 页面名称     | 核心任务                     | 关键交互                         |
| :----------- | :--- | :------- | :----------------------- | :--------------------------- |
| **1. 认知**    | 1    | 欢迎页       | 首次启动，传递价值 + 注册/登录        | 注册/登录后进入个人主页                 |

| <br />      | 11   | 个人穿搭偏好    | 查看历史保存的穿搭方案              | 记录衣物、Tag、推荐理由、日期、场合等         |

### 路由流程

```
Tag1 (欢迎页/注册登录)
  └─ 登录成功 → Tag8 (个人主页)
                  ├─ 头像菜单 → Tag9 (个人档案) / Tag10 (电子衣柜) / Tag11 (穿搭偏好)
                  └─ 开始穿搭 → Tag2 (拍照) → Tag3 (用户档案) → Tag4 (搭配设置)
                                  └─ Tag5 (穿搭结果)
                                      ├─ 满意保存 → Tag12 (新首页)
                                      └─ 不满意 → Tag7 (自由搭配) → Tag12 (新首页)
```

### 新用户跳过限制

- Tag 2（拍照页）和 Tag 3（用户档案页）均设有「跳过」按钮
- **首次注册的新用户**不允许跳过，必须上传衣物 + 完善档案
- **老用户（登录）**可自由跳过
- 判断依据：`window.IsNewUser` 全局标记（注册时设为 `true`，登录时设为 `false`）

## 三、各页面详细设计

### Tag 1：欢迎页（步骤 1）

- **布局**：左上头像，右上注册，中央插画（全息扫描衣橱），主标题 + 副标题，底部主按钮 + 协议小字。
- **交互**：点击主按钮 → 弹出注册/登录面板；登录成功后 → 跳转个人主页（步骤 8）。
- **设计细节**：暖黄渐变圆形底 + 衣架图标，营造温暖科技感。
- **登录逻辑说明**：
  - **账号 ： admin　密码 ： admin123　角色 ：管理员**
  - 登录模式 ：输入 admin / admin123 可成功登录，提示"管理员登录成功"；输入错误会提示"账号或密码错误"
  - 注册模式 ： admin 账号不可重复注册，会提示"该账号已被注册"；其他账号可自由注册
  - 手机号/第三方登录 ：不受影响，仍可直接体验
  - **新用户判断**：注册 → `window.IsNewUser = true`；登录（含 social login）→ `window.IsNewUser = false`
    刷新页面即可在登录弹窗中切到"登录"标签页使用 admin 账号。

### Tag 1.5：个人主页（步骤 8）

- **布局**：左上用户头像（可点击），无右上注册/登录按钮；中央展示今日穿搭或欢迎信息；底部「开始今日穿搭」按钮。
- **头像菜单**：点击头像 → 左侧弹出 3 个圆形按钮，分别指向：
  - **个人档案**（步骤 9）：查看/编辑身材与偏好数据
  - **个人穿搭偏好**（步骤 11）：查看历史穿搭记录
  - **电子衣柜**（步骤 10）：浏览/管理衣物
- **设计细节**：圆形按钮带图标 + 文字，柔和阴影，小红书风格配色。

### Tag 2：拍照识别页（步骤 2）

- **核心**：调用相机/相册，上传图片触发 AI 识别。
- **交互**：选图后「下一步」高亮，点击后不等待识别结果，直接跳转 Tag 3（异步处理）。
- **辅助**：拍照视角选择（平铺/悬挂/上身），默认平铺。
- **跳过按钮**：右上角「跳过→」；新用户显示「跳过🚫」且不可点击，点击后提示"新用户请先上传衣物完善档案，之后即可跳过"。
- **API 标注**：`POST /v1/garments/recognize`（单图） / `POST /v1/garments/segment`（多图）。

### Tag 3：用户档案页（步骤 3）

- **表单内容**：
  - 身体数据：身高、体重、肩宽、体型、日常尺码。
  - 穿搭偏好：风格标签（多选限 3）、颜色偏好（色块多选）、裤长偏好（分段器）。
- **进度条**：假进度条（8 秒），阶段文案（读取→分割→识别→渲染），化解等待焦虑。
- **交互**：点击「查看今日穿搭建议」→ 跳转 Tag 4。
- **跳过按钮**：右上角「跳过」；新用户显示「跳过🚫」且不可点击，点击后提示"新用户请先完善基础档案，之后即可跳过"。

### Tag 4：搭配设置页（步骤 4）

- **参数配置**：
  - 今日场合（通勤/约会/运动/居家/商务/出游）
  - 今日天气（晴/多云/雨/雪）+ 体感温度（凉爽/舒适/偏热）
  - 推荐策略（通勤得体/时尚吸睛/舒适优先/显高显瘦）
  - 搭配画风（日常随拍/杂志大片/通勤风）
  - 高级筛选（色系偏好 / 单品优先级）
- **交互**：右上角「重置」一键恢复默认，高级筛选可折叠。
- **API 标注**：`POST /v1/outfits/recommend`（`application/json`）

### Tag 5：穿搭结果页（步骤 5）

- **内容**：情境标签（天气/场合/风格）、中央虚拟试穿大图、推荐 Tips 卡片、搭配清单（横向滚动）。
- **交互**：
  - 「换一套」：同页切换不同套系。
  - **「满意，保存穿搭」**：弹出命名 Modal → 保存穿搭到 `window.SavedOutfits` → 设为今日穿搭 (`window.TodayOutfit`) → 跳转新首页（步骤 12）。
  - **「不满意，自由搭配」**：直接跳转 Tag 7 自由搭配工坊。
- **按钮设计**：上下排列的紧凑卡片式按钮，渐变紫色保存按钮 + 浅灰边框跳过按钮，各带图标和副标题。
- **API 标注**：`POST /v1/tryon`（异步生成试穿图）+ `POST /v1/tips/generate`（生成推荐理由）

### Tag 7：自由搭配工坊（步骤 7 · 核心）

- **布局**：左侧中央换衣大图（Drop Zone），右侧我的衣柜（分类 Tab + 缩略图网格）。
- **交互流程**：
  1. 用户点击右侧分类 Tab（全部/外套/上衣/下装/裙装/鞋履），分类展开显示对应缩略图。
  2. 用户**拖拽**缩略图到中央大图区域。
  3. 释放后触发 AI 换衣：图片转圈 + 进度条（0%→100%）+ 阶段文案 + 穿搭 Tips 轮播（转移注意力）。
  4. 换装完成，中央大图更新为新的试穿效果。
  5. 点击「保存这套穿搭并设为偏好」→ 弹出命名 Modal → 确认保存 → 跳转新首页（步骤 12）。
- **转移注意力设计**：
  - 进度条分阶段（分析身材→匹配版型→优化色彩→渲染细节）。
  - Tips 轮播（每 2.5 秒切换一条穿搭小贴士）。
- **API 标注**：`POST /v1/tryon`（异步换装）+ `POST /v1/user/save_outfit`（保存穿搭 + 上传 Tag）

### 个人档案（步骤 9 · 子页面）

- **来源**：个人主页头像菜单 →「个人档案」
- **内容**：从 Tag 3 保存的用户数据以小红书风格表单展示，包括：
  - 身体数据：身高、体重、肩宽、体型、日常尺码
  - 穿搭偏好：风格标签、颜色偏好、裤长偏好
- **交互**：只读表单展示，底部「编辑档案」按钮（Demo 中跳回 Tag 3）
- **设计**：左标签右数据，分割线清晰，大量留白，小红书风格。

### 电子衣柜（步骤 10 · 子页面）

- **来源**：个人主页头像菜单 →「电子衣柜」
- **布局**：左侧纵向分类 Tab（全部/外套/上衣/下装/裙装/鞋履/配饰），右侧 2 列网格展示衣物卡片。
- **衣物卡片**：每张卡片包含图标 + 名称 + 颜色 + 季节 + 风格标签（Tag）。
- **交互**：点击左侧 Tab 切换分类，卡片悬停/点击有微动效。
- **示例数据**：16 件预设衣物，覆盖 6 个品类（卡其色风衣、白色衬衫、深灰西裤、棕色乐福鞋、黑色皮衣、格纹衬衫、束脚运动裤等）。

### 个人穿搭偏好（步骤 11 · 子页面）

- **来源**：个人主页头像菜单 →「个人穿搭偏好」
- **内容**：展示从 Tag 5 和 Tag 7 保存的历史穿搭方案，每条记录包含：
  - 穿搭名称、保存日期、场合
  - 当日天气、穿着感受
  - 搭配衣物列表（含图标）
  - 推荐 Tag（风格标签）
  - 推荐理由
  - 来源标记（Tag5 穿搭结果 / Tag7 自由搭配）
- **示例数据**：3 条预设穿搭记录 + 用户保存的动态数据（`window.SavedOutfits`）。
- **交互**：卡片列表滚动浏览，点击「查看详情」可展开。

### 新用户首页（步骤 12）

- **布局**：左上头像（可打开菜单），右上无注册/登录；中央展示今日穿搭图片；下方穿搭描述说明；底部「自由搭配」按钮 + 修改按钮。
- **内容**：
  - 今日穿搭效果展示区域
  - 穿搭说明文字（可修改）
  - 「✂️ 自由搭配」按钮 → 跳转 Tag 7
  - 「✏️ 修改今日穿搭」按钮 → 重新调整
- **设计**：小红书风格卡片布局，柔和色彩，大量留白。
- **来源**：Tag 5 保存穿搭后 / Tag 7 保存穿搭后 → 均跳转到此页面。

## 四、API 接口汇总

| 接口名称    | Method | Path                     | Content-Type          | 用途              | 调用阶段                  |
| :------ | :----- | :----------------------- | :-------------------- | :-------------- | :-------------------- |
| 单品识别    | POST   | `/v1/garments/recognize` | `multipart/form-data` | 识别单件衣物品类/颜色     | Tag 2                 |
| 单品分割    | POST   | `/v1/garments/segment`   | `multipart/form-data` | 分割多件衣物（异步）      | Tag 2                 |
| 穿搭推荐    | POST   | `/v1/outfits/recommend`  | `application/json`    | 基于情境推荐整套穿搭      | Tag 4                 |
| 虚拟试穿    | POST   | `/v1/tryon`              | `application/json`    | 生成上身效果图（异步）     | Tag 5 / Tag 7         |
| 穿搭 Tips | POST   | `/v1/tips/generate`      | `application/json`    | 生成推荐理由文案        | Tag 5                 |
| 风格匹配    | POST   | `/v1/styles/match`       | `application/json`    | 首次用户匹配风格标签      | Tag 5（首次）             |
| 保存穿搭    | POST   | `/v1/user/save_outfit`   | `application/json`    | 保存用户定制的穿搭 + Tag | Tag 7                 |

## 六、交互设计原则

1. **异步不等待**：拍照后立即跳转填表，AI 后台运行，消除等待焦虑。
2. **假进度条 + 阶段文案**：让用户感知 AI 在"分步骤思考"，而非单纯卡顿。
3. **转移注意力**：Tag 7 中通过 Tips 轮播 + 动态微动效，化解换衣等待时间。
4. **流畅动画**：所有页面切换采用 0.38s 滑动过渡（左右滑入滑出），符合移动 App 直觉。
5. **小红书风格表单**：左标签右输入/选择，分割线清晰，大量留白，视觉清爽。
6. **新用户引导**：首次注册用户不可跳过拍照和档案填写，确保数据完整性（`window.IsNewUser` 控制）。

## 八、运行方式

本 Demo 沿用原型的 **React + Babel Standalone（浏览器内编译 JSX）** 方案，无构建步骤。
由于各组件以外部 `<script type="text/babel" src="...">` 形式加载，**必须通过本地 HTTP 服务器访问**（直接双击 `index.html` 会因 `file://` 的 CORS 限制无法加载组件）。

进入 `Veslune-Frontend_DEMO/VestiCore-Prototype/` 目录，任选一种方式启动静态服务：

```bash
# Python（推荐，默认端口 3000）
python -m http.server 3000

# Node (npx)
npx serve . -p 3000
```

然后浏览器打开 `http://localhost:3000/` 即可。

## 九、拆分说明

- `theme.css`：原 `<style>` 块完整迁出，含 `:root` 设计 Token、phone-frame、page-slot、动画类、nav/btn/card/tag/segment/steps/ai-progress/api-hint 等通用组件，以及各页面专用样式（头像菜单、电子衣柜、结果按钮、跳过提示、新首页布局等）、Modal、Toast、响应式。
- `TagX_*.js`：每个页面组件函数原样迁出，并通过 `window.PageX = ...` 暴露到全局，便于跨文件引用。
  - `Tag1_Welcome.js`：`window.PageWelcome`，含注册/登录逻辑和 `window.IsNewUser` 标记。
  - `Tag1_5_Home.js`：`window.PageHome`，个人主页，含头像点击弹出圆形菜单（个人档案/穿搭偏好/电子衣柜），步进到 Tag 2。
  - `Tag2_Scan.js`：`window.PageScan`，含 `hasImage`、`viewAngle`、`skipTip` 三个 useState，跳过按钮受 `window.IsNewUser` 控制。
  - `Tag3_Profile.js`：`window.PageProfile`，含 progress/statusText/form/skipTip 四个 useState + useEffect 模拟 8 秒假进度条，跳过按钮受 `window.IsNewUser` 控制。
  - `Tag4_Settings.js`：`window.PageSettings`，含 settings/collapseOpen useState，今日场合/天气/策略/画风/高级筛选。
  - `Tag5_Result.js`：`window.PageResult`，含 currentIndex/loading/showSaveModal/outfitName/toastMsg useState，3 套穿搭数据。满意→保存穿搭并跳转新首页（步骤 12），不满意→跳转 Tag 7。
  - `Tag7_Workshop.js`：`window.PageWorkshop`（最大组件），含 10 个 useState + useRef，拖拽换衣逻辑、AI 进度模拟、Modal 保存，保存后→新首页（步骤 12）。
  - `ProfilePage.js`：`window.PageProfileDetail`，小红书风格只读表单展示 Tag 3 保存的用户数据。
  - `WardrobePage.js`：`window.PageWardrobe`，左侧分类 Tab + 右侧衣物卡片网格，含 16 件示例衣物 + 6 个品类。
  - `PreferencePage.js`：`window.PagePreference`，展示历史穿搭记录卡片列表（3 条示例 + `window.SavedOutfits` 动态数据）。
  - `NewHomepage.js`：`window.PageNewHome`，展示今日穿搭 (`window.TodayOutfit`)，含头像菜单、穿搭描述修改、自由搭配入口。
- `App.js`：保留原 `AppContext` 定义、`App` 路由函数与 `ReactDOM.createRoot(...).render(<App />)`；路由步骤：1, 2, 3, 4, 5, 7, 8, 9, 10, 11, 12（无步骤 6）。内含 step/direction/prevStep/animating/returnStep 状态管理，goTo/next/back/jumpTo 方法，renderPage/renderPrevPage switch 12 个 case。
- `index.html`：在加载 React CDN 后用一段内联脚本把 `useState / useEffect / useRef / useCallback / createContext / useContext` 挂到 `window`，使各组件函数体内可直接使用这些 Hook。组件按 Tag1→Tag1.5→Tag2→Tag3→Tag4→Tag5→Tag7→ProfilePage→WardrobePage→PreferencePage→NewHomepage→App 顺序加载，确保 App.js 取到所有 `window.PageX`。

原始单文件原型 `VestiCore_demo/VestiCore · 产品原型 Demo.html` 未做任何修改。
